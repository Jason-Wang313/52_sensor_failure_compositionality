# Reviewer Attacks

## 1. The interaction-aware model wins because it is given the true law.

Response: Addressed by v3. The final benchmark includes isolated, additive, pairwise, coverage-unaware high-order, order-calibrated, and oracle protocols. The calibrated protocol is not given the oracle true order and remains below oracle.

## 2. Pairwise laws are not universal.

Response: Sustained and central. V2 hidden higher-order stress remains the negative control. The final claim is order calibration, not pairwise sufficiency.

## 3. The synthetic experiment is too small and stylized.

Response: Addressed in scale but not in deployment. The full-scale suite now has 432000 compact rows representing 99283968000 evaluations, but it remains simulated and deterministic.

## 4. Existing multi-sensor corruption benchmarks already enumerate combinations.

Response: Partly sustained. The novelty is not enumerating combinations alone. The novelty is reporting supported interaction order, coverage, false robust certification, and unsupported abstention.

## 5. The model does not handle common-cause failures.

Response: Addressed but not solved. The benchmark includes sparse high-order common-cause and cascading latent common-cause regimes. The calibrated protocol lowers false robust certification relative to pairwise, but cascading common causes remain a hard boundary.

## Decision Impact

The final framing is submission-ready as a full-scale simulated benchmark and reporting protocol with explicit limits. The paper should keep the oracle upper bound and real-robot limitation visible.
